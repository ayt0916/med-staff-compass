## 2. アプリケーション概要

本アプリケーション「Med-Staff Compass」は、医療現場のスタッフが直面する「業務負荷」と「心理的ウェルビーイング」を可視化・統合分析するためのWebダッシュボードです。

### コンセプト
**「技術的効率化」を「スタッフの幸福度(Well-being)」に確実に結びつける、データ駆動型・現場伴走型プラットフォーム**

医療現場では、業務効率化（例：電子カルテ導入）が、必ずしもスタッフの負担軽減に繋がらず、むしろ「効率化で生まれた時間で、更なる肉体労働（患者ケア）をさせられる」という不信感が存在します。

本アプリは、この課題を解決するため、以下の2種類のデータを同時に可視化します。

1.  **物理的負荷:** 「管理的負荷（カルテ業務など）」と「身体的負荷（患者ケアなど）」のバランス
2.  **心理的負荷:** パルスサーベイに基づくウェルビーイング状態

これにより、経営層・管理者は「効率化が、スタッフの休憩時間確保や心理的安全性の向上に本当に繋がっているか」をデータで確認し、現場の不信感を払拭する（＝質的マンパワーの向上）ための具体的な意思決定を支援します。

---

## 3. 解決したい課題

* **現場の不信感:** 業務効率化によって削減された「管理的負荷」の時間が、単に「身体的負荷（患者ケア）」に振り分けられ、総労働負荷が変わらない（むしろ増える）ことへの不満。
* **人手不足の加速:** 上記の不信感や心理的ストレスが蓄積し、現場スタッフのエンゲージメントが低下。結果として離職に繋がり、人手不足がさらに加速する悪循環。
* **経営層の課題:** 経営層が「効率化＝負担軽減」となっていることをデータで確認できず、スタッフのメンタルヘルス状態も把握できないため、根本的な対策（人員配置の見直し、環境改善）が打てない。

## 4. 主な機能

本アプリケーションは、「スタッフ（個人）ビュー」と「管理者ビュー」の2つの視点を切り替えることができます。

### 共通機能
* **ビュー切り替え:** ヘッダーのトグルスイッチで、「スタッフ（個人）ビュー」と「管理者ビュー」をシームレスに切り替えます。

### スタッフ（個人）ビュー

スタッフ個人が自身の状態を客観的に把握するための機能です。

* **業務負荷バランス（個人）:**
    * 「身体的負荷」「管理的負担」「その他（休憩など）」の割合をドーナツチャートで可視化します。
    * **今月と先月のデータを比較表示**し、特に「管理的負担（削減目標）」と「その他（休憩など、確保目標）」の増減を数値で明示します。
* **心理的ウェルビーイング（個人）:**
    * 「ウィークリー・サーベイ」の結果をレーダーチャートで即時反映します。
* **ウィークリー・パルスサーベイ（入力）:**
    * モーダルウィンドウから「職場のサポート満足度」「業務の評価満足度」「ノンストレス度」の3項目を回答できます。
    * 回答データは即座に個人のウェルビーイングチャートに反映されます。

### 管理者ビュー

管理者（経営層・部門長）がチーム全体の傾向を把握し、意思決定を行うための機能です。

* **チーム全体の業務負荷バランス:**
    * スタッフ個人のデータを集計した、**チーム全体の「業務負荷バランス」**を今月・先月で比較表示します。
    * これにより「チーム全体で管理的負担が削減され、休憩時間が確保できているか」というマクロな視点での分析が可能です。
* **チーム全体のウェルビーイング:**
    * チームメンバーのサーベイ結果の**平均値**をレーダーチャートで可視化します。
    * チーム全体の心理的安全性の傾向を把握します。
* **チーム負荷ヒートマップ (モックアップ):**
    * （将来実装イメージ）「いつ」「誰の」負荷が高いかを可視化し、人員配置の偏りを特定するためのヒートマップです。
* **改善アクション (モックアップ):**
    * （将来実装イメージ）分析データに基づき、「管理的負担とストレスレベルの相関」などを分析し、具体的な改善策（1on1の実施、プロセス見直し等）を提案します。
